§ I

The Methodological Boundary Condition

Current mechanistic interpretability frequently relies on "circuits" and "features"—abstractions that often fail to provide object identity. If two researchers can observe the same subgraph and derive different mental models of the underlying computation without an experimental way to adjudicate between them, the framework is not scientific; it is hermeneutic.

We operate under a strict methodological boundary condition:

"Any claim about a transformer that cannot be expressed in front-to-back transformer dynamics is not an explanation of a transformer."

To satisfy this, we must move from storytelling to system identification using the local geometry of the model's flow field.

§ IV

Disconfirmation Criteria

This framework invites falsification. If the following were true, these results would be invalid:

Criterion A — Null-Model Parity

If a base (un-tuned) or random-weight model shows the same v* norm explosion at Layer 26, the result is an artifact of architecture, not learning.

Criterion B — No Conditioning Signal

If singular value summaries (p>1, p≈1) do not differ between "Verbose" and "Concise" trajectories, the Gain Geometry claim is void.
